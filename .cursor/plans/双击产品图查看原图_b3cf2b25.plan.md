# 双击产品图查看原图大图

## 背景

- 产品图以 120px 缩略图显示在 [ui/product_tab.py](ui/product_tab.py) 的 `ProductCard`（L224）中；`image_label` 设置了 `WA_TransparentForMouseEvents`，鼠标事件由卡片本身接收。
- 数据库只存相对路径 `products.image_path`，原图是磁盘文件，通过 `models.get_product_image_path(product)`（[db/models.py](db/models.py) L844）解析为绝对路径。
- 目前没有任何双击事件或大图预览功能。

## 实现方案

全部改动集中在 [ui/product_tab.py](ui/product_tab.py)：

### 1. ProductCard 增加双击信号

- 新增 `double_clicked = Signal(object)`（携带 product）。
- 实现 `mouseDoubleClickEvent`：左键双击时发射信号。Qt 双击前会先派发一次 `mousePressEvent`（触发选中），行为兼容，无需特殊处理。

### 2. 新增 ImagePreviewDialog 大图对话框

- `QDialog`，标题显示产品名和原图分辨率（如 `卡片A — 1200×1600`）。
- 用 `QPixmap(路径)` 加载原图；若超过屏幕可用区域的约 90%，等比缩小到适配屏幕，否则按原尺寸显示（`KeepAspectRatio` + `SmoothTransformation`）。
- 按 Esc 或点关闭按钮关闭（QDialog 默认支持 Esc）。

### 3. ProductTab 接线

- 在 `refresh_products`（L577–583 建卡处）连接 `card.double_clicked.connect(self._show_original_image)`。
- 新增 `_show_original_image(product)`：用 `models.get_product_image_path(product)` 取路径；文件不存在或 `QPixmap` 加载失败时弹 `QMessageBox.warning` 提示"图片文件不存在或无法加载"，否则打开预览对话框。

## 验证

运行 `py -3.11 main.py`（当前终端已有实例在跑，需先关闭），在产品管理页双击任一产品卡片，确认弹出原图大图；单击选中、右键菜单、拖拽等原有交互不受影响。
