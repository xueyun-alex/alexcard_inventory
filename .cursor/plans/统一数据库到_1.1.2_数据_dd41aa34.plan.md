# 统一数据库:以 1.1.2 数据为准,今后共用一份

## 背景

- 最新最全的数据在 `dist\1.1.2\AlexcardInventory\data\`(今天 17:47 还在更新,860 KB)。
- `py main.py` 用的是仓库根目录 `data\`(停留在 7/10,旧数据)。
- 原因是 [app_paths.py](app_paths.py) 的 `project_root()` 固定向上跳两级找 `main.py`,而打包脚本把 exe 放在 `dist\<版本>\AlexcardInventory\`(三级深),找不到就各自建了独立 `data\`。
- 数据库中图片路径是相对 `data\` 存储的([db/models.py](db/models.py) 第 396 行),因此整个 `data\` 文件夹可以安全整体复制。

## 安全红线(全程遵守)

- 对 `dist\1.1.2\AlexcardInventory\data\` **只做读取和复制,不写入、不删除、不重命名**。
- 现有仓库根目录旧 `data\` 不删除,先重命名备份。

## 执行步骤

### 1. 备份旧数据

把仓库根目录现有 `data\` 重命名为 `data_backup_20260716\`(保留,不删除)。

### 2. 同步 1.1.2 数据到仓库根目录

把 `dist\1.1.2\AlexcardInventory\data\` 完整**复制**(Copy-Item,非移动)到仓库根目录 `data\`,包含 `inventory.db`、`products\`、`trash\`。复制后核对文件数量与字节大小一致。

### 3. 修复 `project_root()` 路径解析

修改 [app_paths.py](app_paths.py):frozen 状态下不再固定跳两级,而是从 exe 目录逐级向上查找含 `main.py` 的目录;找到则共用该目录(即仓库根)的 `data\`,找不到(分发到别人电脑时)仍回退到 exe 所在目录。

```python
if is_frozen():
    exe_dir = Path(sys.executable).resolve().parent
    for parent in [exe_dir, *exe_dir.parents]:
        if (parent / "main.py").is_file():
            return parent
    return exe_dir
```

### 4. 打包新版本 1.1.4

用 `scripts/build_exe.ps1 1.1.4` 打出新版本(不覆盖任何旧版本目录,自然不会触碰 1.1.2)。新 exe 位于 `dist\1.1.4\AlexcardInventory\`,由于步骤 3 的修复,它会自动定位到仓库根目录的 `data\`,与 `main.py` 共用同一份数据库。

### 5. 只读验证

- 运行 `py main.py`,确认产品数、库存、操作记录与 1.1.2 一致。
- 运行 `dist\1.1.4\AlexcardInventory\AlexcardInventory.exe`,确认看到同样的数据。
- 验证后再次核对 `dist\1.1.2\...\data\inventory.db` 的修改时间和大小未变,证明其未被触碰。

## 说明:旧版本 exe 无法重定向

1.1.0–1.1.3 的 exe 是用旧路径逻辑打包的,代码已冻结在 exe 内,不重新打包无法让它们改用共享数据库。它们的目录**原样保留不动**(含各自的旧 data),仅作为历史存档;今后请使用 `main.py` 或 1.1.4 及以后的版本,数据即完全统一。
